Golang: A Brief Overview

  1. Features of Golang:
    • Runtime environment: Manages essential functions like goroutines, garbage collection, and memory allocation
    • Efficient garbage collection: Automatically manages memory, freeing developers to focus on business logic
    • Concurrency approach: Uses lightweight goroutines for efficient concurrent execution
  2. Golang's runtime environment:
    • Compiled into every Go binary
    • Controls goroutines, garbage collection, scheduling, and more
    • Uses environment variables like GOGC, GOMEMLIMIT, and GODEBUG to control runtime behavior
  3. Garbage collection in Go:
    • Automatically finds and frees unused memory
    • Uses a non-generational concurrent, tri-color mark and sweep garbage collector
  4. Concurrency in Go:
    • Achieved through goroutines, which are lightweight and efficient
    • Goroutines start with only 2kB of stack memory and can grow or shrink as needed
    • Communication between goroutines is done via channels
  5. Differences from Java:
    • Go is a general-purpose language, while Java is purely object-oriented
    • Go compiles faster and runs faster than Java
    • Go uses defer, panic, and recover for error handling instead of exceptions
    • Go's concurrency model (goroutines) is more lightweight than Java's threads
    • Go doesn't use private/public keywords; visibility is determined by capitalization
